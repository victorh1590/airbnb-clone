Requisitos do Sistema.
Para instalar o sistema é necessário a utilização do stack LAMPP ou XAMPP.
Foram utilizados no desenvolvimento do sistema as seguintes ferramentas:
->PHP 7.2.24-0ubuntu0.18.04.7 - Zend Engine v3.2.0
->MYSQL Ver 14.14 Distrib 5.7.32, for Linux (x86_64) using EditLine wrapper
->APACHE 2 Server version: Apache/2.4.29 (Ubuntu)
->phpmyadmin
->Opcional: DBeaver, MYSQL Workbench

Configurando o sistema.
Após a instalação do XAMPP ou LAMPP stack é necessário configurar o Apache2 redirecionando-o para a pasta do projeto. No caso do LAMPP stack essa pasta se encontra em /var/www. No caso do XAMPP é provavelmente a pasta htdocs em /opt/lampp/.
Caso a simples transferência da pasta para o local não retorne a página em https://localhost/index.html, o seguinte procedimento deve ser seguido:

https://docs.joomla.org/Configuring_a_LAMPP_server_for_PHP_development/Linux_desktop

No link acima, as partes: Enabling mod_rewrite, Creating the New Site substituindo caminhos da configuração pelo caminho da pasta do projeto, e os 3 comandos:

sudo a2ensite mydevsite
sudo a2dissite 000-default
sudo service apache2 restart

Depois garantir que a pasta do projeto contém as seguintes permissões: Access-> Read&Write para o dono, grupo e outros. As mesmas permissões devem também estar ativadas na pasta images_product, ou o sistema não permitira escrita de novas imagens na pasta.
Em caso de dúvida o seguinte comando pode ser utilizado:

sudo chmod 777 {caminho/da/pasta (ex: /home/clausius/4)} -- Recomendo checar a sintaxe do comando com cuidado, pois o comando chmod pode causar danos sérios ao sistema.

Em seguida configuraremos o banco de dados. Se os passos acima foram seguidos corretamente, você deve ter agora acesso à pasta do projeto pela url https://localhost/index.html. Precisamos rodar a query de criação do banco temvaga_db contida no arquivo temvaga_db.sql na pasta sql. A query pode ser rodada pelo https://localhost/myphpadmin, pelo MYSQL Workbench, pelo DBeaver ou pela interface mysql no terminal. Atenção aos dados do usuário root do sistema (id e senha).

Em seguida rodar a query contida em imoveis.sql dentro do banco temvaga_db para criar a tabela única imoveis. Após ser criada a tabela, pode ser feita uma consulta como SELECT * FROM imoveis; ou verificação pelo phpmyadmin para checar se os campos com entradas que contém acentos (como 'São Paulo') foram criados corretamente. Caso não, significa que a query não criou o banco com o charset UTF-8, e neste caso deletaremos a tabela imóveis e a criaremos novamente usando a query contida no arquivo temvaga_db.sql, que está comentada, rodando também o imoveis.sql em seguida para inserção dos dados.

Encerrada a configuração do banco, precisamos configurar as permissões para o PHP. Para isso, se dirigir para a pasta php, abrir o arquivo db_connection.php e editar as 3 variáveis contidas nele com base na sua máquina: 
          $servername = "localhost"; //Local ou IP do servidor.
          $username = "root"; //Username do admnistrador do BD.
          $password = "@DMin123"; //Senha do admnistrador do BD.
E está terminada a configuração. A página inicial do site é: http://(localhost ou ip)(:porta?)/index.html;
Estrutura de pastas.
        css -> contém o único .css do projeto.
        
        images -> contém imagens de exemplo e elementos do site.
        
        images_product -> 
          contém as imagens do anúncios registrados no banco de dados. 
          NÃO ALTERAR OS ARQUIVOS AQUI MANUALMENTE, APENAS PELO SISTEMA. 
        
        js -> 
          contém o script principal script.js e outros scripts especiais para 
          determinadas páginas.
       
        logs -> não necessária, só um lugar para o apache2 salvar alguns logs de erro.
        
        php -> todos os scripts php (server-side) do site.
       
        sql -> querys para configuração do banco do sistema (que já fizemos).
Arquivos na pasta principal do projeto. 
          about.html -> Esta página Sobre.
          
          cadastro.php -> Página de cadastro de anúncios.
          
          checkin.html -> 
            Página específica de um anúncio. 
            Inacessível sem os parâmetros GET na URL.
          
          controlpanel.php -> 
            Painel de Controle para edição/remoção de registros (anúncios) 
            no banco de dados.
         
          editar.php -> Página de edição de anúncios.
          index.html -> Landing Page do site (HOME).
          products.html -> Página de anúncios/busca de anúncios.
          
          signup_system.php -> 
            Apenas um middleware de validação de sessão para selecionar 
            edição ou cadastro, deve ser inacessível ao usuário comum.
          
          web_gui_kit.html -> WEB GUI KIT do site, nossa identidade visual.
          
          README.txt -> apenas uma repetição desta página.
Onde encontrar os requisitos do trabalho 2?
          Validação Javascript -> Metade final do js/script.js
          
          Preenchimento com Javascript -> 
            Cards na página products.html, 
            trapézio vermelho no banner das páginas index.html e products.html, 
            dados nos formulários da página editar.php (acessível através de controlpanel.php, clikando no lápis verde), 
            informações em todas as páginas checkin.html (acessível pela página products.html, clickando em 'visualizar' de algum anúncio), 
            a tabela de registros em controlpanel.php.
         
            Troca de Palheta -> 
              Qualquer página, exceto o web_gui_kit.html, clikando na lâmpada na header.
Onde encontrar os requisitos do trabalho 3?
          Tela de cadastro (inclusão) -> cadastro.php (https://localhost/cadastro.php)
          
          Remoção de Registros -> 
            No Painel de Controle, 
            controlpanel.html clikando no X vermelho de algum registro. 
            (https://localhost/controlpanel.php)

          Edição de Registros -> 
            No Painel de Controle, 
            controlpanel.html clikando no lápis verde de algum registro. 
            (https://localhost/controlpanel.php)

          Realização de consultas com um ou mais critérios -> 
            Página products.html (https://localhost/products.html)
            Aceita buscas por 'código': númerico.
            Cidade: String sem números.
            Categoria: String (valores pré-definidos que podem ser verificados em cadastro.php).
            Diversos filtros contidos na página. Interagem com o resultado das buscas.
            Para limpar a busca: deixe o campo em branco e aperte enter;
ATENÇÃO: EM CASO DE QUALQUER PROBLEMA NAS PÁGINAS, TENTE DAR REFRESH COM SHIFT + F5 PARA LIMPAR O CACHE.
ATENÇÃO: CASO ALGUM SCRIPT PHP DÊ PROBLEMA, DESCOMENTE O CÓDIGO COM AS VARIÁVEIS $username e $password e mude para as corretas.
ATENÇÃO: O ARQUIVO ZIPADO image_products PODE SER USADO PARA RECUPERAR A PASTA image_products APÓS DELETAR REGISTROS.
